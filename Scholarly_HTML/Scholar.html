1. Abstract

Cal O Web este o aplicatie va permite gestionarea cu usurinta a alimentelor consumate de catre un utilizator. In cadrul proiectului se va construi o aplicatie ce va implementa un sistem client-server ce va fi folosit pentru a expune printr-un site web toate facilitatile acestei aplicatii.

2. Descriere

Aplicatia va permite utilizatorului sa se autentifice sau inregistreze inainte de a avea acces la functionalitatile acestei aplicatii web. Dupa autentificare, utilizatorul poate sa creeze retete proprii sau sa modifice retete existente si sa isi calculeze numarul de calorii mancate ( la o masa, intr-o zi/saptamana/luna ) pentru a-si imbunatati stilul de viata. Retetele acestuia si contul vor fi salvate intr-o baza de date MySQL. Aplicatia va oferi o gama de peste 1600 de produse din care utilizatorul sa isi creeze retete perfecte. Pe pagina de profil a acestuia sunt vizibile: numele, prenumele, email-ul, inaltimea, greutatea si IMC-ul (Indice de Masa Corporala) plus cateva sfaturi utile pentru a-si imbunatati sanatatea si stilul de viata.

3. Componente

3.1 Login / Register

Utilizatorii se vor putea autentifica sau inregistra. Logarea se face pe baza de username/parola iar inregistrarea implica introducerea de date valide pentru username, parola, nume, prenume, inaltime, varsta, greutate si gen. Toate datele vor fi analizate in timp real inainte ca userul sa dea "Submit", astfel evitam reintroducerea inutila a datelor si enervarea clientului.

3.2 Gestionarea retetelor

Odata autentificat, utilizatorul poate crea sau modifica retete care sunt in baza de date, sa isi marcheze anumite retete pe care le-a mancat in decursul timpului. Toate aceste informatii sunt vizibile pe profilul acestuia alaturi de cateva sfaturi legate de sanatatea acestuia.

Pe acest link puteti urmari un tutorial pentru login.

3.3 Calcularea caloriilor

Dupa ce se logheaza un utilizator are acces la butonul care ne va duce catre calculatorul de calorii. Ajuns pe pagina calculatorului are posibiliatea de a importa o reteta deja existenta in baza de date sau adaugarea unui produs din baza de date. Pentru fiecare produs cerut are posibilitatea de a edita sau de a sterge acel produs.

Pe acest link puteti urmari un tutorial pentru calculare.

3.4 Adaugarea unei noi retete

Odata logat acesta va putea accesa pagina de adaugare a unei retete. Aceasta trebuie sa contina un nume valid ( doar litere ) si cel putin un produs. Caloriile retetei vor fi calculate automat si sunt afisate in partea de jos a formularului in care introduceti produsele.

5.2 Baza de date

  1. Tabela cantitati ( id_reteta, id_produs, cantitate ) contine toate produsele pentru fiecare reteta cat si cantitatea din acesta. Este
    folosita pentru a afisa valoarea calorica si ingredientele pe pagina de calculator.
  2. Tabela produse ( id, nume, valaore_calorica ) contine tot sortimentul de produse oferit de aplicatia noastra.
    Este utilizata la adaugarea unei noi retete sau la calcularea caloriilor.
  3. Tabela profile ( id, firstname, lastname, email, inaltime, greutate, varsta, sex ) contine toate datele despre persoanele care au cont la noi, le folosim pentru gestiunea tuturor informatiilor si oferirea unor indicatii.
  4. Tabela retete ( id, nume, id_creator, val_cal ) retinem informatiile mai generale despre retete. Aceste date le folosim in principal pe pagina de profil si la incarcarea produselor sale in calculator.
  5. Tabela retete_urmate ( id_reteta, id_persoana, data_urm ) retinem id-ul retetelor pe care un utilizator le-a mancat si a dorit sa le pastreze. Aceasta tabela este folosita pe pagina de profil pentru a-l personaliza.
  6. Tabela users ( username, password ) retinem contul fiecarei persoane si il folosim la logare si la inregistrare sa verificam daca un username deja exista.

5.3 Entitati

Entitate utilizator -> este imaginea clientului aplicatiei prin aplicatia noastra.
Este formata din:

  • id
  • nume
  • prenume
  • email
  • inaltime
  • greutate
  • varsta
  • sex
Entitate produs -> este produsul real transformat in informatii utile pentru aplicatia noastra.
Este formata din:
  • id
  • nume
  • valoare_calorica
Entitatea reteta -> este o mancare careia i-au fost introduse ingredientele in baza de date si numarul de calorii.
Este formata din:
  • id
  • nume
  • id_creator
  • val_cal
Pentru aceasta entitate produsele sunt stocate in alta tabela.
Entitate user -> este contul prin care un client poate avea acces la aplicatie.
Este formata din:
  • id_persoana
  • username
  • parola
